**Descripción:**
Como administrador del sistema
Quiero buscar logs introduciendo texto libre que coincida con el mensaje del error
Para localizar rápidamente un error concreto del que recuerdo parte del mensaje

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Happy Path):** Dado que escribo "connection refused" en el campo de búsqueda, cuando aplico el filtro, entonces la tabla muestra solo los logs cuyo campo `message` contiene esa cadena (case-insensitive).
- [ ] **Escenario 2 (Sin resultados):** Dado que el texto buscado no coincide con ningún log, cuando se aplica el filtro, entonces se muestra el mensaje vacío sin errores de aplicación.
- [ ] **Escenario 3 (Debounce):** Dado que el usuario escribe en el campo de búsqueda, cuando están pasando menos de 400 ms desde la última pulsación, entonces no se lanza la petición al backend (debounce en frontend).
- [ ] **Requisito Técnico:** Usar `ILIKE '%term%'` en PostgreSQL. NO usar `LIKE` sin índice en tablas grandes. Evaluar `pg_trgm` index si el volumen supera 100k registros. Sanitizar input para evitar wildcards maliciosos.
