# [SHOULD] F-05.3 — Autenticación Básica del Administrador

**Épica:** EPIC-05 — Infraestructura y Conectividad

---

**Descripción:**
Como administrador del sistema
Quiero autenticarme con usuario y contraseña para acceder al panel
Para evitar que cualquier persona con acceso a la red interna pueda ver y gestionar los logs

**Criterios de Aceptación (Checklist):**
- [ ] **Escenario 1 (Login exitoso):** Dado que introduzco las credenciales correctas en el formulario de login, cuando pulso "Entrar", entonces se establece una sesión autenticada y se redirige al dashboard.
- [ ] **Escenario 2 (Credenciales incorrectas):** Dado que introduzco una contraseña incorrecta, cuando pulso "Entrar", entonces se muestra un mensaje de error genérico ("Credenciales incorrectas") sin indicar qué campo falló.
- [ ] **Escenario 3 (Sesión persistente):** Dado que he iniciado sesión y cierro y abro el navegador, cuando vuelvo a la URL del panel, entonces la sesión sigue activa (cookie httpOnly persistente).
- [ ] **Escenario 4 (Logout):** Dado que pulso "Cerrar sesión", cuando se ejecuta la acción, entonces la cookie de sesión se invalida y se redirige al login.
- [ ] **Escenario 5 (Rutas protegidas):** Dado que un usuario no autenticado intenta acceder a cualquier ruta del panel, cuando realiza la petición, entonces es redirigido al login.
- [ ] **Requisito Técnico:** Usar **Laravel Sanctum** en modo SPA (cookie-based, httpOnly, SameSite=Strict). Un único usuario admin configurado en `db:seed` o en `.env`. No exponer endpoint de registro. Proteger contra brute force con throttle en el endpoint de login (STRIDE S-API-01).
