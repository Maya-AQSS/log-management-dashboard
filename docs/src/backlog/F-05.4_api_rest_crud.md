# [MUST] F-05.4 — API REST para CRUD de Logs Archivados y Comentarios

**Épica:** EPIC-05 — Infraestructura y Conectividad

---

**Descripción:**
Como desarrollador del sistema
Quiero que Laravel 12 exponga una API REST clara y segura para todas las operaciones sobre logs archivados y comentarios
Para que el frontend React pueda consumirla de forma predecible y mantenible

**Criterios de Aceptación (Checklist):**
- [ ] **Escenario 1 (Endpoints documentados):** Dado que el proyecto está en desarrollo, cuando se consulta la API, entonces todos los endpoints siguen convenciones RESTful y están documentados (al menos con comentarios en el controlador o colección Postman).
- [ ] **Escenario 2 (Respuestas consistentes):** Dado que el frontend realiza cualquier petición, cuando recibe la respuesta, entonces el formato JSON es consistente: `{ data: ..., meta: ..., errors: ... }` (Laravel API Resources).
- [ ] **Escenario 3 (Errores manejados):** Dado que ocurre un error de validación (422) o de servidor (500), cuando el frontend recibe la respuesta, entonces el cuerpo contiene mensajes de error legibles y estructurados.
- [ ] **Escenario 4 (Autenticación en todos los endpoints):** Dado que cualquier endpoint de la API es llamado sin sesión válida, cuando se procesa la petición, entonces se devuelve 401 Unauthorized.
- [ ] **Requisito Técnico:** Endpoints mínimos a implementar: `GET /api/logs`, `GET /api/logs/{id}`, `POST /api/logs/{id}/archive`, `GET /api/archived-logs`, `GET /api/archived-logs/{id}`, `POST /api/archived-logs/{id}/comments`. Todos protegidos con `middleware:auth:sanctum`. Usar Laravel FormRequest para validación. Usar Laravel API Resources para serialización.
