**Descripción:**
Como administrador del sistema
Quiero que el editor de comentarios en el histórico sea el mismo editor rich text usado al archivar
Para poder añadir seguimiento con imágenes y formato al igual que el comentario inicial

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Consistencia visual):** Dado que abro el editor para añadir un comentario al hilo, cuando lo visualizo, entonces tiene las mismas capacidades de formato que el editor del archivado inicial (F-03.3).
- [ ] **Escenario 2 (Imágenes en seguimiento):** Dado que pego una captura de pantalla en el editor del hilo, cuando guardo el comentario, entonces la imagen queda embebida y visible en el hilo.
- [ ] **Escenario 3 (Límite aplicado):** Dado que intento subir una imagen mayor de 2 MB en un comentario de seguimiento, cuando se intenta insertar, entonces se muestra el mismo error que en F-03.3.
- [ ] **Requisito Técnico:** Reutilizar el mismo componente TipTap 2 de F-03.3. Aplicar idénticas reglas de sanitización y límites de tamaño (STRIDE T-RTE-01, D-RTE-01, E-RTE-01).
